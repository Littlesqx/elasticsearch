[[mapping-types]]
== 字段数据类型

Elasticsearch 中文档的字段支持很多不同的数据类型：

[float]
=== 核心数据类型

string::        <<text,`text`>> 和 <<keyword,`keyword`>>
<<number>>::    `long` ，`integer`, `short`, `byte`, `double`, `float`, `half_float`, `scaled_float`
<<date>>::      `date`
<<boolean>>::   `boolean`
<<binary>>::    `binary`
<<range>>::     `integer_range`, `float_range`, `long_range`, `double_range`, `date_range`

[float]
=== 复杂数据类型

<<array>>::     数组的支持不需要一个专用的 `type` 。
<<object>>::    简单的 JSON 对象。
<<nested>>::    JSON 对象的数组。

[float]
=== 地理数据类型

<<geo-point>>::     经纬度坐标。
<<geo-shape>>::     复杂图形，例如多边形。

[float]
=== 专业数据类型

<<ip>>::            IPv4 或 IPv6 地址。
<<search-suggesters-completion,Completion datatype>>::
                    `completion` 类型提供自动补全建议。
<<token-count>>::   `token_count` 类型可统计字符串中的词元数量。
{plugins}/mapper-murmur3.html[`mapper-murmur3`]:: `murmur3` 在索引时计算一些值的哈希值并将它们存储在索引中。
{plugins}/mapper-annotated-text.html[`mapper-annotated-text`]::  `annotated-text` 用于索引包含特殊标记的文本（通常用于识别命名实体）。

<<percolator>>::    接受来自 query-dsl 的查询。

<<parent-join>>::   为相同索引内的文档定义父/子关系。

<<alias>>::         为已存在的字段定义别名。


[float]
=== 多字段

为了达到不同的目的，以不同的方式索引相同的字段通常是有用的。
例如，字符串字段可以被<<mapping-index,索引>>为用于全文搜索的文本字段，并作为用于排序或聚合的关键字字段。
或者，可以使用<<analysis-standard-analyzer,标准分析器>>，
<<analysis-standard-analyzer,英文分析器>>和<<french-analyzer,法语分析器>>等对字符串字段进行索引。

这是 _多字段_ 的目的，而大多数数据类型可通过 <<multi-fields>> 参数支持多字段。
